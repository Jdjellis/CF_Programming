# Daily adjust — output format

A mid-week readiness adjustment re-emits only the **affected day(s)**, briefly.
Map the athlete's report to a tier (policy §2), restate the changed sessions with
re-run loads, and give a one-line reason. Keep it phone-length — no re-listing the
whole week.

**Rules in one line:** AMBER = keep each PROTECT top set, trim back-off ~half,
autoregulate CRUISE (−1–2 RPE / fewer rounds). RED = drop loaded PROTECT/CRUISE to
skill / active recovery; SKILL survives; ACCESSORY → rehab/mobility only.

---

## Example — Thu @ 🟡 AMBER

**Thu 11 Jun · class: gymnastics**

- **🔴 PROTECT — Strict Press** (kept, back-off trimmed)
  - Top single @ RPE 8 → `58 kg` (83% of 70) ⭐
  - Back-off ~~3×5~~ **2×5** @ 75% → `52.5 kg`
- **🟡 CRUISE — gym WOD:** autoregulate −1–2 RPE, trim a round; keep quality.

---

## Example — Tue @ 🔴 RED

**Tue 9 Jun · class: engine**

- **🟡 CRUISE — engine WOD → active recovery:** skip loaded work; easy aerobic /
  mobility only.
- **🟣 ACCESSORY — quad/knee → rehab only:** Spanish-squat holds + knee work; drop
  the loaded split/zombie squats.

> Loaded work dropped to skill / active recovery (policy §2). Re-run `calc.py` for
> any changed target; honour the warm-up read over a wearable score and note any
> override.
